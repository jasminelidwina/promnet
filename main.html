Git Topic

Version Control System (VCS)

Version Control System adalah sebuah sistem yang digunakan dalam pengembangan perangkat lunak untuk melacak dan mengelola perubahan dalam kode sumber dan dokumen proyek sepanjang waktu. Tujuan utama dari VCS adalah untuk mengelola versi berbeda dari file dan mengkoordinasikan kolaborasi antara tim pengembang yang bekerja pada proyek yang sama.

Git

  • Git adalah sistem kontrol versi distribusi yang digunakan untuk melacak perubahan dalam kode sumber proyek perangkat lunak bebas dan open source.
  • Git digunakan untuk mengendalikan versi kode sumber dengan melacak perubahan, membuat cabang (branches), dan memeriksa riwayat perubahan.
  • Git diinstal dan dijalankan pada komputer lokal sehingga tidak memerlukan koneksi internet untuk berfungsi.
  • Git tidak menyediakan fasilitas hosting untuk repository kode sumber. Pengembang biasanya harus membuat repository Git di komputer mereka sendiri atau menggunakan server Git yang tersedia.

Github

  • GitHub adalah sebuah layanan berbasis web untuk proyek-proyek yang menggunakan Git untuk pengendalian versinya dan menyediakan fitur kolaborasi, seperti issue tracking, code review, dan fitur-fitur lain untuk tim.
  • GitHub menyimpan repositori Git di cloud, berkolaborasi dengan tim lain, dan membagikan pekerjaan dengan komunitas.
  • GitHub merupakan layanan berbasis web/cloud yang dapat diakses dari mana saja dengan koneksi internet.
  • GitHub dirancang untuk memudahkan kolaborasi dengan mengundang kontributor, mengatur hak akses, dan melihat kontribusi dari orang lain melalui pull requests.

Git Bassic Command

  • git init: Inisialisasi repository Git baru di direktori saat ini. Ini membuat repository kosong dan memungkinkan memulai melacak perubahan kode.
  • git add [nama file]: Menambahkan file atau perubahan ke "staging area", yang merupakan persiapan untuk commit. Misalnya, git add file.txt akan memasukkan file.txt ke dalam staging area.
  • git commit -m "[pesan commit]": Membuat commit dengan pesan deskriptif yang menjelaskan perubahan yang telah dilakukan.
  • git status: Menampilkan status file dalam repository, seperti file yang diubah, yang sudah di-add, atau yang belum di-add.
  • git pull: Mengambil perubahan terbaru dari repository yang sudah ada (biasanya dari server jarak jauh) dan menggabungkannya dengan branch saat ini.
  • git push: Mengirim perubahan lokal ke repository yang sudah ada (biasanya ke server jarak jauh) untuk menyimpan perubahan tersebut.
  • git branch: Menampilkan daftar branch yang ada dalam repository, dengan branch saat ini dicetak tebal.
  • git checkout [nama branch]: Beralih ke branch lain. Misalnya, git checkout master akan beralih ke branch "master."
  • git remote: Menampilkan daftar remote repository yang terhubung ke repository lokal.
